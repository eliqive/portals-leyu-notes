# portals-leyu-notes

本仓库用于归档和发布多个独立 HTML 页面。这些页面以静态方式存储，不依赖特定后端服务，可直接通过浏览器访问或部署到任意静态托管平台。

## 项目简介

该仓库主要作为 HTML 页面的集中存放与版本管理场所。每个页面独立存在，彼此之间无强关联，适用于记录、展示或分发各类静态内容。

## 目录说明

```
.
├── index.html          # 首页 / 导航页（可选）
├── pages/              # 存放各独立 HTML 页面
│   ├── page-001.html
│   ├── page-002.html
│   └── ...
├── assets/             # 资源文件（图片、样式、脚本等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/` 目录下存放所有归档的 HTML 文件，文件名遵循 `page-xxx.html` 或自定义名称。
- `assets/` 目录用于存放页面引用的静态资源，按类型分文件夹管理。
- 首页 `index.html` 可根据需要提供页面索引或导航功能，非必需。

## 页面归档说明

- 每个 HTML 页面均为独立完整文件，包含其所需的全部样式与脚本（或通过相对路径引用 `assets/` 中的资源）。
- 页面之间无跨域依赖，可直接打开或部署后访问。
- 归档内容不针对任何特定域名或网站，仅作为静态内容存档。

## 维护说明

- 新增页面：在 `pages/` 目录下创建新的 HTML 文件，并确保其资源引用路径正确。
- 更新页面：直接修改对应 HTML 文件，建议在提交信息中注明变更内容。
- 删除页面：移除相关文件，并检查 `index.html` 或导航页中是否仍存在指向该页面的链接。
- 请保持目录结构整洁，避免放置无关文件。

## 部署

本仓库可直接用于 GitHub Pages 或其他静态托管服务。将仓库设置为公开后，在仓库设置中启用 Pages 功能，选择 `main` 分支作为源，即可通过 `https://<用户名>.github.io/portals-leyu-notes/` 访问。
